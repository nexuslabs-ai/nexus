Focus-ring policy comparison

Each option below shows the actual rendered focus state under both light and dark themes, using the real OKLCH values from packages/tailwind/. The button itself uses the brand primary color in each mode.

Decision · 2026-05-21 · ✅ Option D selected & implemented

Option D shipped in PR #55 · the "requires re-tuning" caveat was addressed

All 8 component focus sites now use nx:focus-visible:shadow-focus-default. The token's color was retuned from the near-white starting values (oklch(0.9219 0 0 / 0.502)) to solid mid-grey — oklch(0.5555 0 0) light, oklch(0.7572 0 0) dark — clearing WCAG 2.2 SC 1.4.11 (APCA Lc ≥ 45).

Designer pushed matching Figma values 2026-05-21: focus.color.default → #737373 / #b0b0b0; focus.color.error → #dc2626 / #fca5a5. Figma↔code parity for focus closed via #56 + PR #65.
