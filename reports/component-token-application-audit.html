Nexus Component Token Application Audit

The component layer is mostly healthy after the brand and control-token work. The remaining improvements are about stricter token ownership: lines should use border tokens, neutral selected states should use control tokens, popover interactions should use popover tokens, and passive illustration/chrome can keep muted.

Audit Rubric

Surface Tokens

Use background, container, popover, and nav-background for real planes. These are not interchangeable once base modes shift.

Control Tokens

Use control-background for switch rails, tab rails, sliders, progress tracks, and neutral selected states.

Border Tokens

Use border-default for frames and border-default-alpha for soft 1px separators. Avoid using muted as a divider color.

Text And Status

Keep muted-foreground for secondary copy. Use primary/status tokens only for brand actions and real status meaning, not ordinary informative chrome.

Cleanup Decisions

1. Menu Separators

DropdownMenu, ContextMenu, Menubar, Select, and Command use bg-muted for 1px separators. Change those to bg-border-default-alpha.

2. Tooltip Coloring

Tooltip previously used primary-background. With real pink/orange/purple brands, that made tooltips brand-colored. Move it to a compact popover surface with a readable border.

3. Popover Row States

Menu items inside bg-popover panels previously used background-hover. Use popover-hover with popover-foreground instead.

4. Neutral Selection

Table selected rows use bg-muted. That state is selected, not passive. control-background is a better neutral selection token.

5. Dialog Close State

Dialog and Sheet close buttons use inert data-state=open:bg-muted classes. Remove those and add explicit hover/focus-visible affordances.

6. Story TODO Cleanup

Button, Badge, and Alert stories still contain old WCAG contrast TODOs. The repo is APCA-gated, so these comments are stale and violate the tracked-TODO convention.
